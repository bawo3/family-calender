# 📅 캘린더 일정표

브라우저에서 바로 사용할 수 있는 단일 파일 캘린더 일정 관리 앱입니다.
모든 일정 데이터는 브라우저의 **로컬스토리지(localStorage)** 에 저장되어 새로고침이나 브라우저를 다시 열어도 그대로 유지됩니다.

---

## 🚀 사용 방법

1. `index.html` 파일을 더블클릭하여 브라우저에서 엽니다.
2. 별도의 서버나 설치 과정 없이 바로 사용할 수 있습니다.

---

## ✨ 주요 기능

### 1. 월별 캘린더 보기
- 현재 월의 달력이 표시됩니다.
- 일요일은 **빨간색**, 토요일은 **파란색**으로 구분됩니다.
- 오늘 날짜는 **파란색 테두리**로 강조됩니다.

### 2. 월 이동
| 버튼 | 기능 |
|------|------|
| ◀ 이전 | 이전 달로 이동 |
| 다음 ▶ | 다음 달로 이동 |
| 오늘 | 현재 월로 즉시 이동 |

### 3. 일정 추가
1. 원하는 날짜 셀을 클릭합니다.
2. 하단의 입력창에 일정 내용을 입력합니다.
3. **추가** 버튼을 누르거나 **Enter** 키를 누르면 저장됩니다.

### 4. 일정 조회
- 각 날짜 셀에 등록된 일정이 **주황색 박스**로 미리보기 표시됩니다.
- 한 날짜에 일정이 3개 이상이면 `+N` 형태로 추가 개수가 표시됩니다.
- 날짜를 클릭하면 하단 패널에 해당 날짜의 모든 일정이 표시됩니다.

### 5. 일정 삭제
- 하단 일정 목록에서 각 일정 옆의 **삭제** 버튼을 클릭하면 즉시 삭제됩니다.

---

## 💾 데이터 저장 방식

### 저장소
- 브라우저의 **localStorage** API 사용
- 키 이름: `calendar_events`

### 저장 형식 (JSON)
```json
{
  "2026-05-07": ["회의", "약속"],
  "2026-05-15": ["생일파티"]
}
```
- 키: `YYYY-MM-DD` 형식의 날짜 문자열
- 값: 해당 날짜의 일정 텍스트 배열

### ⚠️ 주의사항
- localStorage는 **브라우저별, 도메인별**로 분리되어 저장됩니다.
- 다른 브라우저(예: Chrome → Edge)에서는 데이터가 공유되지 않습니다.
- 브라우저의 캐시/사이트 데이터를 삭제하면 일정도 함께 사라집니다.
- 시크릿 모드에서는 창을 닫으면 데이터가 사라집니다.

## 🛠️ 기술 스택

| 분류 | 사용 기술 |
|------|-----------|
| 마크업 | HTML5 |
| 스타일 | CSS3 (Grid 레이아웃) |
| 로직 | Vanilla JavaScript (라이브러리 없음) |
| 저장소 | Web Storage API (localStorage) |

## 🔑 핵심 함수 설명

| 함수 | 역할 |
|------|------|
| `loadEvents()` | 로컬스토리지에서 일정 데이터를 불러옴 |
| `saveEvents(events)` | 일정 데이터를 로컬스토리지에 저장 |
| `formatDate(year, month, day)` | 날짜를 `YYYY-MM-DD` 문자열로 변환 |
| `renderCalendar()` | 현재 월의 캘린더 그리드를 그림 |
| `selectDate(dateStr)` | 사용자가 클릭한 날짜를 선택 상태로 변경 |
| `renderEventList()` | 선택한 날짜의 일정 목록을 표시 |
| `addEvent()` | 새 일정을 추가하고 저장 |
| `deleteEvent(idx)` | 인덱스로 일정을 삭제 |

---

## 💡 사용 팁

- 입력창에서 **Enter** 키를 누르면 빠르게 일정을 추가할 수 있습니다.
- **오늘** 버튼으로 언제든지 현재 월로 빠르게 돌아올 수 있습니다.
- 한 날짜에 여러 일정을 등록할 수 있으며, 캘린더 셀에는 최대 2개까지 미리보기가 표시됩니다.
